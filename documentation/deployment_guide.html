False Alert Suppression: Detailed Deployment Guide

This document provides a comprehensive, step-by-step guide to deploying the False Alert Suppression pipeline, the background consumer, the API, and the React Traceability Dashboard in a production environment.

1. System Prerequisites & Dependencies

  • Python 3.10+: Required for LangGraph, FastAPI, and the Consumer daemon.
  • Node.js (v18+) & NPM: Required to build and run the React Traceability Dashboard.
  • RabbitMQ Server (v3.10+): Required for the Dead-Letter Exchange (DLX) delay queue architecture. Must have the Management Plugin enabled for UI configuration.
  • MongoDB Server (v5.0+): Required to persist alert execution payloads and historical states.
  • ServiceNow Instance: Must have the standard Table REST API enabled with an active service account.

2. Infrastructure Setup

RabbitMQ DLX Configuration

Important: Service accounts typically do not have configure permissions. You must manually create the queue structures before starting the Python services.
  1. Log into the RabbitMQ Management UI (typically http://localhost:15672).
  2. Navigate to the appropriate Virtual Host (e.g., /noops_automation).
  3. Create Exchange: Go to Exchanges -> Add a new direct exchange named dnac.exchange (Durable).
  4. Create Active Queue: Go to Queues -> Add a queue named dnac.alerts.delayed.q (Durable).
  5. Bind Queue: Click on dnac.alerts.delayed.q -> Bindings -> Bind to dnac.exchange with routing key dnac.alerts.delayed.
  6. Create Delay/Wait Queue: Go to Queues -> Add a queue named dnac.alerts.wait.q (Durable). Expand arguments and add:
    • x-dead-letter-exchange = dnac.exchange
    • x-dead-letter-routing-key = dnac.alerts.delayed
    • x-message-ttl = 900000 (15 minutes in milliseconds)

3. Application Installation

A. Python Backend Environment

# 1. Navigate to the project root
cd /path/to/false-alert-suppression

# 2. Create and activate a virtual environment
python -m venv venv
# On Windows: venv\Scripts\activate
# On Linux: source venv/bin/activate

# 3. Install required Python packages
pip install langgraph pika requests python-dotenv pyyaml pymongo fastapi uvicorn

B. React Dashboard Environment

# 1. Navigate to the dashboard directory
cd dashboard

# 2. Install NPM dependencies
npm install

4. Credentials & Configuration (.env)

Create a file named .env in the root directory (false-alert-suppression/.env) and populate it with the following required credentials. Do NOT commit this file to version control.

# ==========================================
# 1. LangGraph & ML
# ==========================================
OPENAI_API_KEY=sk-your-openai-api-key-here

# ==========================================
# 2. RabbitMQ Connection
# ==========================================
RABBITMQ_USERNAME=svc_rabbitmq_noops_consumer
RABBITMQ_PASSWORD=your_secure_rmq_password

# ==========================================
# 3. ServiceNow Integrations
# ==========================================
SNOW_INSTANCE_URL=https://dev12345.service-now.com
SNOW_USERNAME=svc_snow_automation
SNOW_PASSWORD=your_secure_snow_password

# ==========================================
# 4. MongoDB Database
# ==========================================
MONGO_URI=mongodb://localhost:27017
MONGO_DB_NAME=false_alert_suppression

# ==========================================
# 5. Cisco DNAC
# ==========================================
DNAC_USERNAME=svc_dnac_reader
DNAC_PASSWORD=your_secure_dnac_password
Tip: Ensure your config.yaml in the root directory correctly maps the RabbitMQ host and DNAC base URL.

5. Running the System (Execution Guide)

To run the complete system, you must start four separate processes (typically via systemd, pm2, or Docker containers in production).

Process 1: The FastAPI Backend Bridge

This serves MongoDB and live ServiceNow data to the React Dashboard.

cd /path/to/false-alert-suppression
source venv/bin/activate
uvicorn dashboard.api:app --host 0.0.0.0 --port 8000

Process 2: The Delayed Queue Consumer

This daemon runs infinitely, listening for expired alerts from RabbitMQ and orchestrating the ServiceNow escalation.

cd /path/to/false-alert-suppression
source venv/bin/activate
python consumer/consumer-delay-queue.py

Process 3: The React Traceability Dashboard

This provides the UI for engineers.

cd /path/to/false-alert-suppression/dashboard
npm run build
npm run preview -- --host 0.0.0.0 --port 3000
# (Or serve the /dist folder via Nginx/Apache in production)

Process 4: Alert Ingestion (The Producer)

When DNAC triggers an alert webhook, execute the LangGraph pipeline.

cd /path/to/false-alert-suppression
source venv/bin/activate
python workflow/run.py
